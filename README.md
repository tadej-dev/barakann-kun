## 1. サービス概要（※最後に書いてOK）

ロードバイクのバラ完（パーツを個別に選んで組み上げること）を支援するシミュレーターサービスです。

フレーム・コンポーネント・ホイールなどを自由に組み合わせ、価格・重量を可視化することで、初心者から上級者まで安心して理想の1台を設計できる環境を提供します。

ロードバイクユーザーが抱える「この組み合わせで本当に組めるのか？」「完成したらいくらになるのか？」という不安を解消することを目的としています。

---

## 2. このアイデアはどこから生まれたか

ロードバイクを趣味として続ける中で、いつか自分だけのバラ完車を組みたいと考えていました。
しかし実際に情報収集を始めると、

- フレームとBB規格が合うのか
- ホイールやローターの規格は対応しているのか
- 完成時の重量や総額はいくらになるのか

など、確認すべき情報が非常に多くありました。

結局、メーカーサイト・ショップサイト・ブログ・YouTube・SNSを何度も行き来しながらExcelで管理することになり、「もっと簡単に全体を把握できる方法はないのか」と感じました。

---

### 2-2. なぜそれが「気になった」のか

私は何かを購入する前に比較検討することが好きで、納得した上で意思決定したいタイプです。
また、ロードバイクは高額な趣味であり、パーツ選びの失敗が大きな損失につながります。

そのため、
「完成してから失敗に気づく」
のではなく、
「購入前に判断材料を揃えたい」
という価値観が強くありました。

しかし、その判断を支援してくれるサービスが見当たらず、不便さを強く感じたことが開発のきっかけです。

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

バラ完に必要な情報が複数のサイトに散らばっており、価格・重量をまとめて確認できない。

---

### 3-2. 本当に解決したい課題は何か

なぜ困るのか？
→ パーツ同士が適合するか分からない。

なぜそれが困るのか？
→ 購入後に組み合わせられない可能性がある。

なぜそれが困るのか？
→ 高額な買い物で失敗する不安が大きく、理想のバイク作りに踏み出せない。

→ 本当に解決したい課題：
「ユーザーが失敗への不安なく、自信を持って理想のロードバイクを設計・購入できる状態を作ること」

### 10-2. 技術選定の理由

なぜこの技術を使うのか：
JavaおよびSpring Bootの学習を継続しており、バックエンド開発への理解を深めたいと考えているためです。
また、本サービスではパーツ情報や互換性判定など、複雑なデータ構造や業務ロジックを扱うことが想定されます。
Spring Bootは保守性や拡張性に優れており、将来的な機能追加にも対応しやすいと考えています。

今回チャレンジしたい点：

- Spring Securityを利用した認証機能の実装
- JPAを利用した複雑なデータモデリング
- パーツ互換性判定ロジックの設計
- 検索機能の実装
- 将来的なSPA構成を見据えたAPI設計

不安な点：

- ドメイン設計が複雑になる可能性がある
- 互換性判定ロジックの実装難易度が高い
- パーツデータ数の増加によるパフォーマンスへの影響

---

### 10-3.キャッチアップ不足の懸念

Spring Bootを利用した開発経験がまだ十分ではないため、認証機能やデータベース設計、デプロイ作業などで学習コストが発生する可能性があります。

また、パーツ管理や互換性判定機能は要件が複雑であり、実装難易度や調査工数を正確に見積もることが難しいと考えています。

まずはMVPとして、

- ユーザー登録
- パーツ選択
- 重量計算
- 価格計算

に機能を限定し、早期に動くものをリリースします。

その後、利用者のフィードバックをもとに、

- 互換性判定
- パーツ比較
- AIレコメンド

などの機能を段階的に追加する予定です。

また、ReactやNext.jsについては本リリース後にキャッチアップを進め、必要性や開発コストを検証した上で導入を判断します。

#### Spring Boot

以下については学習済みです。

- CRUDアプリケーションの作成
